# Board Summary

## Proposed Product

Virtual Classroom Attendance Using Face Recognition is a web-based system that helps educational institutions automate attendance in live and recorded online classes. It centralizes attendance records, reduces manual effort, and improves auditability for teachers, administrators, and management.

## Why The Board Should Care

- Manual attendance is slow, inconsistent, and easy to manipulate.
- Institutions need better reporting, traceability, and transparency.
- A centralized platform creates operational value beyond attendance, including reporting, compliance support, and digital academic records.

## MVP Scope

The attached product starter already covers:

- User login and role-based access foundation
- Teacher, class, student, and session management
- Attendance record generation workflow
- Reporting dashboard for monitoring participation
- Data model aligned with the proposal's ERD and module structure

## Delivery Approach

The most practical rollout is a phased one:

1. Deploy the attendance management platform first.
2. Validate data flow, user adoption, and reporting quality.
3. Add face registration, image encoding, and video ingestion.
4. Enable automated recognition after privacy and infrastructure approval.

## Risks And Mitigation

- Privacy risk:
  Store facial data with explicit consent, retention policy, and role-based access.
- Accuracy risk:
  Begin with manual review and confidence thresholds before full automation.
- Infrastructure risk:
  Use recorded video processing first if live processing hardware is limited.
- Compliance risk:
  Review institutional and local biometric data policies before production use.

## Recommendation

Approve the project as a phased digital attendance platform with face recognition as a controlled enhancement, not as a day-one dependency. This framing lowers risk and makes the proposal easier for directors to support.
